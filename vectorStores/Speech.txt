# 🔎 Similarity Search in AI

## 📌 Definition

**Similarity search** is a method used to find items (text, images, vectors, etc.) that are **most similar** to a given query based on meaning, not exact keywords.

It works using **vector embeddings** 🧠.

---

# 🧠 Core Idea

Instead of matching words like:

> “AI engineer jobs”

It matches **semantic meaning**:

> "machine learning developer careers"

Even if the words differ, the meaning is similar.

---

# ⚙️ How It Works (Step-by-Step)

## 1️⃣ Convert Data to Embeddings

Text → Numerical vector (e.g. 768 dimensions)

Example:

```txt
"Artificial Intelligence is powerful"
→ [0.012, -0.444, 0.982, ...]
```

---

## 2️⃣ Store in Vector Database

Examples:

* FAISS
* Chroma
* Pinecone
* Qdrant

---

## 3️⃣ Query Comes In

User types:

> "How strong is AI?"

Convert this to embedding.

---

## 4️⃣ Compute Similarity

The system calculates distance between vectors using:

* **Cosine Similarity** ⭐ (most common)
* Euclidean Distance
* Dot Product

Closest vectors = Most relevant results.

---

# 📐 Cosine Similarity Formula

[
similarity = \frac{A \cdot B}{||A|| \cdot ||B||}
]

Result:

* `1` → Identical meaning
* `0` → Unrelated
* `-1` → Opposite meaning

---

| Use Case               | Example                           |

| Search engines         | Semantic search                   |

---

# 🧠 In Your FAISS + Ollama Setup

When you do:

```python
db.similarity_search("What is AI?")
```

Behind the scenes:

1. Query → embedding
2. Compare to stored embeddings
3. Return top-k closest chunks

That’s similarity search.

---

# 🎯 Why It Is Powerful

Traditional search:

```
Keyword-based ❌
```

Similarity search:

```
Meaning-based ✅
Context-aware ✅
Works with GenAI ✅
```

---

# 🚀 Example in LangChain

```python
docs = db.similarity_search("Give me summary", k=3)

for doc in docs:
    print(doc.page_content)
```

This retrieves the 3 most semantically similar chunks.
